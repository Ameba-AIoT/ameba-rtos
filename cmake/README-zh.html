1. 基本说明

1.1 文件介绍

  • make/global_define.cmake: 定义了常用全局变量, 如各种路径
  • make/utility.cmake: 定义了工具类接口
  • make/flags: 定义了编译链接选项
    • make/common: 定义了通用编译链接选项(所有platform和mcu共用)
    • make/[amebadplus|amebasmart]: 按不同platform区分的编译链接选项
    • make/flags/[km0|km4|kr4|ca32]: 按不同mcu的区分的编译链接选项
  • componet/build_info: 定义各组件的编译信息: 源文件/头文件路径/编译选项
    • 包含dependency_common.cmake, dependency_platform.cmake, dependency_mcu.cmake三个文件
    • 注意: 这部分内容后续会按照组件进行拆分

1.2 组件编译

  • 原组件makefile所在目录会使用CMakeLists.txt替代, 如下是release版本源码可见性不同的两个组件的CMakeLists.txt示例
  • at_cmd, 该组件release时源码可见(不需要生成静态库到asdk/lib/soc or asdk/lib/application)
ameba_target_add(at_cmd                                             # target名称
    p_WRAP_TARGET_NAME                                              # wrap target名成, 如at_cmd_${PROJECT_NAME}, 防止target重名, 比如km0/km4可能都有at_cmd组件
    p_PREFIX lib_                                                   # 设置target前缀为lib_, 缺省时为lib(和cmake默认行为一致)
    p_LINK_LIBRARY                                                  # 设置target要连接的库
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_INCLUDE_DIRECTORIES          # 这是一个interface target, 其包含了全局定义的头文件搜索路径, 链接该target即继承全局头文件搜索路径, 对应原makefile中MODULE_IFLAGS
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_LINK_OPTIONS                 # 这是一个interface target, 其包含了全局定义的链接选项, 链接该target即继承全局链接选项
        src_${c_MCU_PROJECT_NAME}_at_cmd                                # 这是一个interface target, 其包含了at_cmd的源文件/连接选项/头文件路径/宏定义/
    p_APPEND_TO_LIST                                                # 将该target追加到指定列表中
        g_${c_MCU_PROJECT_NAME}_IMAGE2_LIBS                             # 列表名称, 后续该列表中的target都会被链接到image2
    p_DEPENDENTED_BY_TARGETS                                        # 该target被哪些target依赖
        dep_${c_MCU_PROJECT_NAME}_at_cmd                                # 被依赖target名称
)
  • cmsis_dsp, 该组件release时源码不可见, 需要生成静态库到asdk/lib/application
ameba_target_add(cmsis_dsp                                          # 同上
    p_WRAP_TARGET_NAME                                              # 同上
    p_STRIP_DEBUG                                                   # 使能对lib执行objcopy -g操作
    p_ENABLE_DETERMINISTIC_ARCHIVES                                 # 使能对lib执行objcopy -D操作
    p_OUTPUT_PATH ${c_SDK_LIB_APPLICATION_DIR}                      # target输出到指定目录下(asdk/lib/application)
    p_OUTPUT_NAME cmsis_dsp                                         # 最终生成的lib名称
    p_PREFIX lib_                                                   # 最终生成的lib前缀
    p_LINK_LIBRARY                                                  # 同上
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_INCLUDE_DIRECTORIES      # 同上
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_LINK_OPTIONS             # 同上
        src_${c_MCU_PROJECT_NAME}_cmsis_dsp                         # 同上
    p_DEPENDENTED_BY_TARGETS                                        # 同上
        dep_${c_MCU_PROJECT_NAME}_cmsis_dsp                         # 同上
)
  • 注: ameba_target_add详细参数定义及在make/utility.cmake中

2. 最佳实践

2.1 更新: 源文件/头文件路径/编译选项

  • 在对应组件的CMakeLists.txt中找到ameba_target_add函数p_LINK_LIBRARY参数下src_开头的target名称(如src_${c_MCU_PROJECT_NAME}_at_cmd)
  • 该target即定义了该组件的私有编译参数
  • 在component/build_info目录下搜索该target(如src_${c_MCU_PROJECT_NAME}_at_cmd), 找到其所在文件(如dependency_common.cmake)
  • 然后在对应的区块内更新即可, 相关函数如下:
  • 添加源文件:
    ameba_target_sources(src_${c_MCU_PROJECT_NAME}_at_cmd p_SCOPE interface
        # ...
    )
  • 添加头文件搜索路径:
    ameba_target_include(src_${c_MCU_PROJECT_NAME}_at_cmd p_SCOPE interface
        # ...
    )
  • 添加预定义:
    ameba_target_definitions(src_${c_MCU_PROJECT_NAME}_at_cmd p_SCOPE interface
        __RTOS__ # 注意不用加-D, 也可以放到下面的ameba_target_compile_options中, 需要加-D
        NDEBUG
        CONFIG_BUILD_SECURE=1
    )
  • 添加编译选项:
    ameba_target_compile_options(src_${c_MCU_PROJECT_NAME}_at_cmd p_SCOPE interface
        -Wno-unused-parameter                               # 对c/c++源文件都生效
        $<$<COMPILE_LANGUAGE:C>:-Wno-pointer-to-int-cast>   # 仅对c源文件生效
        $<$<COMPILE_LANGUAGE:CXX>:-Wno-literal-suffix>      # 仅对c++源文件生效
    )
  • 上述函数还有扩展函数, 如结合条件判断(详见cmake/utility.cmake), 例如当CONFIG_BT打开时添加源文件:
   ameba_target_sources_if(CONFIG_BT src_${c_MCU_PROJECT_NAME}_at_cmd p_SCOPE interface
       ${c_CMPT_AT_CMD_DIR}/atcmd_bt_ext.c
   )

2.2 新增组件

假设组件名称为foo

step1 注册组件名称

  • 在camke/private_define.cmake中p_dependency_list_full中添加组件名称foo
  • 在camke/private_define.cmake中, 根据组件属性情况添加至如下三个list中
    • p_dependency_list_non_loader: 非loader image(image2)的组件 通常是这个
    • p_dependency_list_loader: loader image(image1)的组件
    • p_dependency_list_secure: secure image(image3)的组件
  • 在cmake/global_define.cmake中定义组件路径的变量(方便引用)
ameba_set(c_CMPT_FOO_DIR  ${c_COMPONENT_DIR}/foo)

step2 定义组件dep

  • 在component/build_info/dependency_common.cmake中添加一个library: dep_${c_MCU_PROJECT_NAME}_foo
add_library(dep_${c_MCU_PROJECT_NAME}_foo INTERFACE)
  • 如果组件包含头文件路径供全局使用(给其他组件用), 则添加
ameba_target_include(dep_${c_MCU_PROJECT_NAME}_foo p_SCOPE interface ${c_CMPT_FOO_DIR}/include)
  • 如果组件release时不对外公开源码, 只生成lib, 需要添加lib依赖
ameba_target_link_if(dep_${c_MCU_PROJECT_NAME}_foo p_SCOPE interface
    ${c_SDK_LIB_APPLICATION_DIR}/lib_foo.a # 这里名称及路径要和下面step4一致
)

step3 定义组件src

  • 在component/build_info下新建文件foo.cmake, 格式可以参考build.cmake
if (__INCLUDED_FOO_BUILD_CMAKE__)  # 改成FOO
    return()
else()
    set(__INCLUDED_FOO_BUILD_CMAKE__ TRUE)
endif()

# 定义组件的编译target, 以src开头, 如果组件分多个子组件的话可以定义多个(参考`network.cmake`)
add_library(src_${c_MCU_PROJECT_NAME}_foo INTERFACE)

# 这里可以参考`component/build_info/dependency_common.cmake`中其他已有组件定义
ameba_target_set(src_${c_MCU_PROJECT_NAME}_foo p_SCOPE interface
    p_COMPILE_DEFINES
        __RTOS__        # 这里添加预定义
    p_COMPILE_OPTIONS
        -O2             # 这里添加编译选项
    p_INCLUDES
        ${c_CMPT_FOO_DIR}/include   # 这里添加头文件路径, 这里可以直接使用cmake/global_define.cmake中定义的路径
    p_SOURCES
        ${c_CMPT_FOO_DIR}/foo.c     # 这里添加源文件
)
  • 在component/build_info/dependency_common.cmake中添加
include(${CMAKE_CURRENT_LIST_DIR}/foo.cmake)

step4 在具体IC项目下使用

  • 比如用于smart ca32, 在smart下project_ap/asdk/make目录下创建文件夹foo
  • 在foo下面添加CMakeLists.txt, 内容参考1.2, 如下是一个不生成app lib的例子
ameba_target_add(foo
    p_WRAP_TARGET_NAME
    p_PREFIX lib_
    p_LINK_LIBRARY
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_INCLUDE_DIRECTORIES
        c_PROJ_${c_MCU_PROJECT_NAME_UPPER}_LINK_OPTIONS
        src_${c_MCU_PROJECT_NAME}_foo   # 这里要和step3中target名称一致(src开头)
    p_APPEND_TO_LIST
        g_${c_MCU_PROJECT_NAME}_IMAGE2_LIBS
    p_DEPENDENTED_BY_TARGETS
        dep_${c_MCU_PROJECT_NAME}_foo # 这里要和step2中target名称一致(dep开头)
)
  • 在amebasmart_gcc_project/project_ap/asdk/CMakeLists.txt中添加子目录
ameba_add_subdirectories(
    make/foo
)
  • 在amebasmart_gcc_project/project_ap/mcu_project_config.cmake添加模块开关
ameba_set(c_DEP_FOO ON) # 注: 这里名称要和step2中一致, 使用全大写foo, c_DEP_

step5 提交

  • 涉及更新文件有:
    • camke/private_define.cmake
    • cmake/global_define.cmake
    • component/build_info/dependency_common.cmake
    • amebasmart_gcc_project/project_ap/asdk/CMakeLists.txt
    • amebasmart_gcc_project/project_ap/mcu_project_config.cmake
  • 涉及新增文件有:
    • project_ap/asdk/make/foo/CMakeLists.txt
    • component/build_info/foo.cmake
